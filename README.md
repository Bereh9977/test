# Package-Manager

1) Проєкт використовує Maven, тому всі залежності завантажуються командою mvn.
Потрібно, щоб було встановлено:
- Java JDK 17 або новіша
- Apache Maven 3.9+

Для завантаження залежностей виконайте команду в корені проєкту (за допомогою cmd):
- mvn package

2) Запуск виконується за допомогою командного рядку, для цього відкрийте консоль в місці, де знаходиться pom.xml, та виконайте команду:
   java -jar target/PackageManager-1.0-SNAPSHOT.jar data/<Назва>.txt

3) Для перевірки стилю коду використовується Checkstyle та тип кодування Google Style, відкрийте cmd в проекті, поряд з файлом pom.xml та виконайте команду:
- mvn checkstyle:check

4) Для запуску тестів відкрийте cmd в проекті, поряд з файлом pom.xml та виконайте команду:
- mvn test

5) Для створення звіту про виконання тестів відкрийте cmd в проекті, поряд з файлом pom.xml та виконайте команду:
- mvn verify

6) Для запуску програми в контейнері Docker необхідно спочатку зібрати Docker-образ. Для цього відкрийте командний рядок у корені проєкту, де знаходиться Dockerfile, та виконайте команду:
- docker build -t package-manager .

Після успішної збірки образу можна запустити програму всередині контейнера з передачею вхідного файлу через монтування тому (volume). Для цього виконайте команду у корені проєкту, де знаходиться Dockerfile:

- docker run --rm -v %cd%\data:/data package-manager /data/<Назва>.txt

Параметр -v використовується для підключення локальної папки data до контейнера як /data, що дозволяє передати вхідний файл без необхідності перебудови Docker-образу.
