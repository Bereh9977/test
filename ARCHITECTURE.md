## Варіант завдання: 
Симулятор системи залежностей (Dependency resolver).

## Середовище виконання:
Для виконання роботи обрав мову програмування Java з архітектурою Meavan.

## Етап 3:
Діаграма компонентів та діаграма класів: https://drive.google.com/file/d/1hyP1GuUrKtdMBH3Jlpuiil_v_jKz37jn/view?usp=sharing

## Діаграма компонентів:
Включає в себе 6 компонентів, а саме Main - точка входу в програму, InputParser - зчитування текстового файлу, ValidationEngine - перевірка циклів, 
ResolverEngine - сортування пакетів, встановлення залежностей (Typological sort), ReportGenerator - вивід пакетів та їх залежностей та циклів,
DependencyGraph - граф, який зберігай пакети та їх залежності.
Два інтерфейси, один з цих це реалізація читання файлу, та другий це генерація звіту.

## Потік даних:
Потік даних у системі починається з компонента Main, який отримує шлях до вхідного файлу та передає його компоненту InputParser.
InputParser зчитує текстовий файл, парсить опис пакетів і їх залежностей та формує структуру DependencyGraph.
Побудований граф передається до ValidationEngine, який виконує перевірку на наявність циклічних залежностей.
У разі відсутності циклів ResolverEngine використовує DependencyGraph для обчислення порядку встановлення пакетів за допомогою топологічного сортування.
Результати перевірки та сортування передаються до ReportGenerator, який формує текстовий звіт і виводить його користувачу.

## Діаграма класів:
Клас PackageId містить атрибути name та version і використовується для ідентифікації пакета.
Клас DependencyGraph зберігає залежності у вигляді структури Map<PackageId, List<PackageId>> та надає методи для додавання пакетів/залежностей і отримання списків залежностей.
Клас InputParser читає вхідний файл та формує об’єкт DependencyGraph.
Клас ValidationEngine виконує перевірку графа, зокрема виявляє циклічні залежності.
Клас ResolverEngine виконує топологічне сортування графа та повертає порядок встановлення пакетів.

## Опис структури даних:
Оскільки система працює із залежностями пакетів, основною структурою даних буде спрямований граф. Найзручніше представити його як Map.
Ключем виступає пакет (наприклад, Package), а значенням — список пакетів, від яких він залежить. Цей список зручно зберігати у вигляді динамічного масиву List, оскільки кількість залежностей може змінюватися.
Для реалізації топологічного сортування:
- HashSet — для зберігання вже відвіданих пакетів;
- Stack — для формування порядку встановлення пакетів після обходу їх залежностей.
